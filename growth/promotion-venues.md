# libre.academy — Promotion Venue Guide

A vetted, paste-ready playbook for promoting **libre.academy** (free, open-source, MIT-licensed, in-browser learn-to-code app). Venues are grouped by category. Dead, pay-to-play, and forbid-self-promo venues are dropped or labeled. Verified June 2026.

**Legend for "Who posts":** `AGENT (GitHub PR)` = an automated agent can open a pull request • `AGENT (no-auth API)` = automatable via an open endpoint • `YOU` = a human must post manually (web form, social, OAuth, or editorial).

> Note: zero genuine no-auth submission endpoints exist across all venues — every non-PR venue requires a human (account, OAuth, or editorial review). Only two venues are agent-submittable, both via GitHub PR.

## What the agent can submit

Only **two** venues accept fully automated submission, both via GitHub pull request. There are **no genuine no-auth API endpoints** anywhere in this list.

### Target 1 — `EbookFoundation/free-programming-books` (PRIMARY, highest value)

- **File:** `more/free-programming-interactive-tutorials-en.md`
- **Section:** `### Language Agnostic` (libre.academy is multi-language)
- **Exact entry line to add:**
  `* [Libre.academy](https://libre.academy) - JavaScript, TypeScript`
- **Placement:** alphabetical within the section. Current order is CodeCombat, Codility, Introduction to the Coding Interview Prep Algorithms, Python Tutor, The Fullstack Tutorial for GraphQL. Insert **immediately above** the `* [Python Tutor](http://pythontutor.com)...` line (becomes the 4th entry — "Libre" sorts after "Introduction...", before "Python Tutor").
- **Format rules:** bullet `* `, no space between `]` and `(`, ` - ` (space-dash-space) before the languages, HTTPS, no emojis, no tracking codes/shorteners.
- **Contributing:** `CONTRIBUTING.md` — file ONE focused single-addition PR; do not bundle. PR title e.g. "Add Libre.academy to Interactive Tutorials (Language Agnostic)".
- **Acceptance gate:** site must be genuinely free (no trial/paywall/account wall) with a valid HTTPS cert.

### Target 2 — `libreprojects/libreprojects`

- **File:** `js/data.json` (branch `master`)
- **Important corrections (verified against live file):** the `category` value is **lowercase** (`"knowledge"`, not "Knowledge") and license slugs are **lowercase** (`"mit"`, not "MIT"). The earlier guidance using "Knowledge"/"MIT" would not match the slugs defined at the top of the file.
- **Exact entry to add** (append into the Knowledge category array, alphabetically among the "L" entries):
  `{"id":"libreacademy","category":"knowledge","name":"Libre Academy","description":"Learn to code","address":"https://libre.academy/learn","introduction":"Interactive learn-to-code app — 90+ courses across 26 languages, runs fully in-browser with no account or install.","licenses":["mit"],"tags":["education","programming"],"source":"https://github.com/InfamousVague/Libre.academy"}`
- **Required fields:** id, category, name, description, address, introduction, licenses, source. `description` must be very short (<~20 chars, shown on logo hover); longer text goes in `introduction`.
- **Logo:** also add `logos/libreacademy.png` — square transparent PNG, ideal 160x160 (non-square art resized to 128x128 centered on a 160x160 canvas).
- **Contributing:** `README.md` / `CONTRIBUTING.md` — fork, edit `js/data.json`, optionally open a tracking issue first; commit message first line: `New project: Libre Academy #<issue-number>`. Add alphabetically within the category.

### No-auth endpoints
None. Every other venue requires a human (account, OAuth session, editorial review, or invite).

## You post (high priority first)

1. **Hacker News — Show HN** — highest-leverage single post. Make /learn try-able with zero signup; factual title; never solicit votes.
2. **r/SideProject** — self-promo-friendly, high fit; post what/why/stack + live link.
3. **Product Hunt** — coordinated launch day; age a maker profile first.
4. **AlternativeTo** — exact "Codecademy alternative" search intent; durable SEO.
5. **Peerlist Launchpad** — strong dev/designer audience, free weekly launch.
6. **DevHunt** — dev-tools-only board, favors open-source.
7. **dev.to** — write a genuine build-log/tutorial (recurring).
8. **Uneed / TinyLaunch / Fazier / MicroLaunch** — free launch boards; backlinks/SEO (one pass each).
9. **LibHunt + Free Software Directory + SaaSHub + StackShare** — low-effort OSS/dir listings.
10. **freeCodeCamp News / HackerNoon / Hashnode** — long-form editorial (effort-gated).
11. **OER Commons / MERLOT** — educator/credibility listings.
12. **Lobsters / Fosstodon** — only if you get an invite and genuinely participate.
13. **Social channels (X #buildinpublic, Bluesky, Mastodon, LinkedIn, Quora, Discord, Pinterest, Shorts/TikTok/Reels)** — ongoing relationship/content channels, not one-shot listings.
14. **Class Central / MOOC List / BetaList** — low-priority pitches; likely rejection on fit.

## Don't bother / red lines

**Pay-to-play (dropped):**
- **OpenAlternative** (openalternative.co) — submit funnel is paid-only ($97/$137/$197); free path is uncertain "slow lane". Perfect topical fit but don't pay; the GitHub repo is an auto-generated README, so no PR path exists.
- **OpenSourceAlternative.to** — free = 6+ month waitlist; $29 to skip. Also fit-mismatch (requires self-hosted alternative to a named product).
- **Startup Stash**, **launchdirectories.com / "submit your SaaS" lists** — pay-to-play directory farms, low relevance.
- **Launching Next** — low relevance; skip unless you specifically want one more do-follow backlink.

**Dropped — forbids self-promo / can't self-submit:**
- **freeCodeCamp Forum** — guidelines explicitly forbid posting your own project. (Use freeCodeCamp News instead.)
- **Slant** — explicit conflict-of-interest policy bars the maker from adding their own product; only organic community inclusion is allowed.
- **CodeNewbie Community** — invite-only and mid-platform-reorg; not openly self-submittable. (Use dev.to with #codenewbie instead.)
- **Open Source Friday** — contributor-recruitment program, not a discovery directory; no list file to PR. Only useful to recruit contributors.
- **Saylor Academy** / **SideProjectors** — low relevance / not a real submission path.

**Hard red lines (don't do this):**
- **No automating community posts** (Reddit, HN, Discord, Mastodon, X, etc.) — this is spam, gets flagged/banned, and sinks the project. Humans post these.
- **Never solicit upvotes or "booster" comments** on HN/Product Hunt/Reddit — detection penalizes the launch.
- **No fake reviews, fake accounts, or astroturfing** anywhere.
- **Don't pay directory fees** for a free MIT project — the free venues above cover real reach.
